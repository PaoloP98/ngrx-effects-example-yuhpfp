GET API DATA will dispatch an Action of type ApiGetMockData that will be handled in './store/effects' service and will map the action to another action of type ApiSuccess (the mock data will success always) after calling successfully the MockApiService

GET API DATA with ERROR instead will dispatch an Action of type ApiGetMockDataWithError that will be handled in './store/effects' service and will map the action to another action of type ApiError because we simulate an Exception inside my MockApiService

Inside src/app/store/effects.ts the behaviour of multiple effects in the same time can be changed using different RXJS operators between mergeMap, switchMap, concatMap, exhaustMap. You can test it by clicking the buttons multiple times and by watching the log results.
